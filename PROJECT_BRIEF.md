\# PROJECT\_BRIEF.md



\## 项目名称



InkScope：手写交互式 AI Canvas iOS 原型



\## 项目目标



本项目希望设计并实现一个可在 iPad 端运行的手写交互式 AI 工具原型。



核心目标是让用户能够通过 Apple Pencil 或手写笔与 AI 进行交互，使用户和 AI 的交互过程更加可视化、可操控、可追踪。



本项目不仅是一个普通聊天工具，而是一个基于手写白板、Agent 角色、内容节点和可视化连接关系的 AI 协作系统。

\- 苹果无边记 Freeform

\- 白板类手写工具

\- Canvas 类 AI 工具



\## 页面整体布局



页面底部有一条悬浮工具栏，用户可以选择不同类型的手写工具，例如：



\- 钢笔

\- 圆珠笔

\- 铅笔

\- 橡皮

\- 马克笔

\- 移动工具

\- 文字输入

\- 语音输入



页面左上方有三个主要入口：



\- Content 栏

\- Agent 栏

\- 设置键



页面左侧有 Agent 栏，里面包括用户自定义的 Agent 角色，例如：



\- 助理

\- 技术

\- 执行

\- 知识

\- 产品



用户可以自定义不同 Agent 的名称、颜色和角色能力。



\## 核心交互：手写内容与 Agent 协作



用户可以在白板上手写内容。



用户写完内容后，可以选择某个 Agent 角色，并用笔将 Agent 连接到手写内容上。



系统需要识别用户的手写内容，并调用对应的 Agent 进行搜索、分析或生成。



Agent 生成的内容会以新的内容框形式出现在白板上。



新生成的内容框需要通过连接箭头与原始内容连接，表示这个内容是由某个 Agent 基于某个上下文生成的。



\## 初始操作流程



1\. 用户在白板上手写内容。

2\. 用户选择一个 Agent。

3\. 用户用笔将 Agent 与手写内容连接。

4\. 系统识别手写内容。

5\. 系统调用对应 Agent 进行处理。

6\. 系统在白板上生成新的内容框。

7\. 新内容框与原始内容之间显示连接箭头。



\## 基础操作功能



\### 1. 派生



用户可以对生成内容继续扩展。



例如：用户选中某个已生成内容，调用某个 Agent 继续生成新的相关内容。



每次派生都需要生成新的内容节点，并通过箭头与来源节点连接。



\### 2. 剪枝



用户可以删除不需要的内容。



剪枝功能用于删除用户认为错误、偏离主题或不需要的内容，从而保持 AI 对话上下文的清晰。



剪枝后，这部分内容不再作为后续 AI 生成的上下文。



\### 3. 嫁接



用户可以把多个节点内容聚合为新的生成来源。



不同 Agent 生成过程中的内容可以通过连接箭头互相连接。



用户可以将两个或多个 Agent 生成的内容进行比较、融合，然后再生成新的内容。



嫁接的目标是从多个节点中生成新的综合内容。



\### 4. 嗅探



用户可以在页面上悬停或查看当前上下文的相关信息。



系统可以根据当前节点，显示上下文、来源、关联内容或潜在延展方向。



\## 产品意义



本项目希望让用户和 AI 的交互过程更加可视化、可操控、可自行管理上下文。



用户不只是和 AI 进行线性对话，而是在一个开放白板中与 AI 共同思考、共同发散、共同整理内容。



系统需要帮助用户管理 AI 生成过程中的上下文关系，避免传统 Chat 类 AI 对话中上下文混乱、内容不可追踪、生成路径不清晰的问题。

\## 开发注意事项



\- 不要把本项目做成普通聊天网页。

\- 不要只做一个输入框和回复框。

\- 页面重点是白板、手写、Agent、节点和连接关系。

\- 交互过程要尽量可视化。

\- 所有生成内容都应该能追踪来源。

\- 用户应该能够管理、删除、连接、扩展 AI 生成内容。

\- 界面风格应参考 iPad 端生产力工具，保持简洁、轻量、现代。
